## About

Explanations - How the code works.

This means the code is described in a way that is simple and covers this concept only. 
It describes the choice of " why has it been done this way ?" and " how does it achieve its goal ?".

It ==IS== a description of ==how== the code functions and ==why== it does.

It is ==NOT== a guide on how to use the code. 
Is is ==NOT== examples of where / when or how its used.


Good explanations go into detail of how the code functions and why it does it, providing enough detail to understand only this concept.
It should only use the information necessary to get this concept across.

It should expect the reader to know any other concepts mentioned inside of the description for this concept.
It should not side-tangent at all into explaining a different concept, but may mention and refer to it. 

It should ideally using technical jargon, but not confusing language.
It should not be verbose or hard to understand.

They should explain any trade-offs or "sacrifices" made.
