## About

Tutorial - Teaching how to use aspects of the code

It gives an example of how to use the code and follow instructions, to solve a complex problem.
It answers " how do i use the code to make a concept ?"

It ==IS== a detailed explanation of a complex problem, and how to use the code to solve it
It is ==NOT== a detailed explanation of what the code does.


A good tutorial should be clear, like a recipe book.
They explain things/concepts in the most minimal and simple way that is easy to understand.
They should focus on concrete steps and not be abstract

They should make the user learn by doing and not just reading.
They should help the user get started on the journey.
The starting instructions can be "hand-holdey", as the user may not be familiar with the project or concept.

They should be reliably repeatable.
