## About

Guides - How to use te code to solve a problem

This means giving examples of where the code is used and how it is applied in a larger scope.
It answers " how is the code used ?" and " how does it fit in with other code ?".

Is ==IS== a usage on ==how== to use the code and ==where== it is used.
It is ==NOT== a description of how the code works on a deep level.


Good guides show how to use the code whilst giving an example of its use.

It may briefly go over what the code does. ( e.g This functions adds two numbers )
It should not tell the user how it works.
It should assumes the user doesn't need to know how it works.

Examples should be simple in scope, and not complex where possible.
Example should try to solve one, or not many problems.
Guides should be clear to follow, showing all steps necessary.
